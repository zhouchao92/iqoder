# 模拟面试题生成

## 触发条件

当用户要求根据简历生成面试题、模拟面试准备、面试问题生成时触发此技能。

## 工作流程

### 第1步：获取简历内容

用户可通过三种方式提供简历：

**方式A：指定目录路径**
用户提供包含简历文件的目录路径，使用 Glob 工具扫描该目录下的简历文件：
- 支持格式：`.pdf`、`.docx`、`.doc`、`.md`、`.txt`、`.html`
- 扫描命令示例：`Glob("resume-dir/**/*.pdf")`、`Glob("resume-dir/**/*.md")` 等
- 找到多个简历文件时，列出文件列表让用户选择目标简历
- 读取文件内容：
  - `.md` / `.txt` / `.html`：直接使用 Read 工具读取
  - `.pdf`：使用 Read 工具读取（支持图片格式的PDF）
  - `.docx`：使用 Read 工具读取

**方式B：在线简历链接**
用户提供在线简历的 URL（如个人网站、LinkedIn、牛客、掘金、知乎、GitHub Profile、在线简历平台等），使用 WebFetch 工具抓取页面内容：
- 使用 `WebFetch(url, query="简历内容")` 获取页面正文
- 若页面需要登录或无法访问，提示用户改用其他方式
- 若页面内容包含多段经历，完整提取所有项目和技术栈信息

**方式C：直接粘贴内容**
用户直接粘贴简历文本内容。

### 第2步：解析简历关键信息

从简历中提取以下关键信息：

| 维度     | 提取内容                                 |
|----------|------------------------------------------|
| 技术栈   | 编程语言、框架、中间件、数据库、云服务等 |
| 项目经历 | 项目名称、业务领域、个人职责、技术亮点   |
| 工作经验 | 工作年限、职位、行业背景                 |
| 教育背景 | 学历、专业                               |
| 其他亮点 | 开源贡献、技术博客、证书等               |

### 第3步：确认生成参数

向用户确认以下参数（使用 AskUserQuestion 工具）：

- **问题数量**：默认 20 道，用户可自定义
- **问题类型分布**（四类均包含，按比例分配）：
  - 技术深度题（约 30%）：针对简历中提到的技术栈深入提问
  - 项目业务题（约 30%）：基于项目经历的业务理解和技术方案提问
  - 场景设计题（约 30%）：结合项目场景的系统设计问题
  - 行为面试题（约 10%）：团队协作、问题解决、项目推进等

### 第4步：生成面试题

根据提取的简历信息，生成面试题。每道题的问题与答案分离，答案使用 `<details>` 折叠隐藏，默认收起：

```markdown
### 问题 N：[题目标题]

- **类型**：技术深度 / 项目业务 / 场景设计 / 行为面试
- **关联内容**：简历中对应的技术栈或项目经历
- **难度**：⭐ / ⭐⭐ / ⭐⭐⭐

**问题描述：**
[具体问题]

<details>
<summary>查看答案与考察要点</summary>

**考察要点：**
- 要点1
- 要点2

**参考答案：**
- 关键回答方向1
- 关键回答方向2

</details>
```

#### 各类型题目生成规则

**技术深度题：**
- 针对简历中列出的每项核心技术栈至少生成 1-2 道题
- 涵盖原理、底层机制、性能优化、常见问题
- 示例：简历提到 Redis → 问 Redis 持久化策略、集群方案、缓存穿透/雪崩处理

**项目业务题：**
- 针对每个重点项目生成 2-3 道题
- 涵盖业务理解、技术选型理由、核心难点、解决方案
- 示例：简历提到电商订单系统 → 问订单状态机设计、分布式事务处理

**场景设计题：**
- 基于项目场景延伸出设计类问题
- 涵盖系统架构、高并发、高可用、扩展性设计
- 示例：简历提到消息队列 → 问如何设计一个保证消息不丢失的队列系统

**行为面试题：**
- 基于项目经历设计 STAR 类问题
- 涵盖团队协作、冲突解决、技术决策、项目推进
- 示例：简历提到跨团队协作 → 问如何处理跨团队技术方案的冲突

### 第5步：输出面试题

将生成的面试题保存为 Markdown 文件：

**输出路径：** `/output/interviews/<简历文件名>-<当前日期>.md`

**输出格式模板：**

```markdown
# 模拟面试题

> 基于简历：[简历文件名]
> 生成日期：[YYYY-MM-DD]
> 题目总数：[N] 道

---

## 一、技术深度题（[n] 道）

### 问题 1：[题目标题]

- **类型**：技术深度
- **关联内容**：xxx
- **难度**：⭐⭐

**问题描述：**
[具体问题]

<details>
<summary>查看答案与考察要点</summary>

**考察要点：**
- ...

**参考答案：**
- ...

</details>

---

### 问题 2：...

...

## 二、项目业务题（[n] 道）
...

## 三、场景设计题（[n] 道）
...

## 四、行为面试题（[n] 道）
...
```

## 注意事项

- 问题必须紧扣简历内容，不要生成与简历无关的通用面试题
- 难度应覆盖初/中/高三个层次，以中高级为主
- 技术深度题应关注候选人实际使用的技术，而非泛泛而谈
- 项目业务题应关注候选人在项目中的实际贡献和决策
- 如果简历信息不足以生成某类题目，向用户说明并调整比例
